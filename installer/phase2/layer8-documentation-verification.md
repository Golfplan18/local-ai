2. Generate the README at `[workspace]/docs/README.md`:

   ```markdown
   # Your Local AI System

   ## Starting
   Double-click the [LocalAI] icon on your Desktop.
   Your browser will open to http://localhost:5000 — your AI chat interface.

   ## Stopping
   Run the stop script at [workspace]/stop.[sh|bat],
   or close the terminal window that appeared when you started.

   ## How This Works

   Your AI interface runs at localhost:5000 — a small server on your
   machine that keeps Python in the loop between you and the AI model.

   This matters because your AI can use tools — web search, file access,
   knowledge search — and those tools run in Python. The browser interface
   at localhost:5000 IS the orchestrator interface. Tool calls execute
   automatically, invisibly, before you see the final response.

   Do not use claude.ai, ChatGPT, or Gemini directly for work that requires
   tools. Those interfaces have no Python in the loop. Use localhost:5000.

   ## Your System Files

   Your AI system is controlled by specification files, not code.

   - **boot.md** ([workspace]/boot/boot.md): The active system specification.
     Your AI reads this file as its operating instructions. To upgrade
     your system, replace this file with a newer version from the
     frameworks directory.

   - **knowledge/** ([workspace]/knowledge/): Put files here that you want
     your AI to be able to search. Notes, documents, project files —
     anything you'd like the AI to reference when answering questions.

   - **logs/** ([workspace]/logs/): Session logs are saved here automatically.
     Each session creates a timestamped file (YYYYMMDD-HHMMSS-description.md).
     These are searchable in future sessions.

   - **frameworks/** ([workspace]/frameworks/): The complete library of
     frameworks from the book. Your AI can execute any of these.

   - **config/endpoints.json**: Lists your available AI connections.
     The orchestrator reads this to know which models are available.

   ## Your Model
   Model: [name]
   Parameters: [count]
   RAM usage: ~[X] GB
   [Brief plain-language description of what this model is good at.]

   ## Commercial AI Access

   Your system accesses commercial AI through browser automation.
   Connected services: [list]

   To add more services or reconnect after a session expires:
   Tell your AI: "Read and execute frameworks/browser-eval-setup.md"

   ## API Keys (optional, for later)

   API keys are the overflow/reliability channel — for when browser
   automation is throttled, for enterprise users who want SLAs, or
   for autonomous tasks that need API access.

   The setup framework is already installed at:
   [workspace]/frameworks/api-key-setup.md

   Tell your AI: "Read and execute frameworks/api-key-setup.md"

   ## If Something Goes Wrong
   - If the browser opens but shows "No AI endpoints configured": the
     endpoint registry may need to be rebuilt. Run this framework again.
   - If the browser opens but shows "connection refused": the server
     isn't running. Click the launcher.
   - If tool calls don't execute (you see <tool_call> tags in the response):
     you may be connected to a commercial AI directly rather than going
     through localhost:5000. Always use the launcher.
   - If the local model responds with garbled text: the chat template
     may need updating. Run this framework again.

   ## Updating Frameworks

   To get the latest versions of all frameworks:
   1. Tell your AI: "Update the framework library"
   2. Or run manually: cd [workspace]/frameworks/book && git pull
   ```

3. Perform final end-to-end verification:
   a. Execute the launcher.
   b. Wait for the browser to open to localhost:5000.
   c. Send a test message through the browser interface.
   d. Verify a response is received and coherent.
   e. Send a second test message requiring web search.
   f. Verify the tool executes and returns a real result.
   g. Execute the stop script.
   h. Verify clean shutdown.

4. Additional verification steps:
   a. Verify Python packages: `python3 -c "import chromadb; import keyring; from playwright.sync_api import sync_playwright; print('OK')"`
   b. Verify orchestrator tools: run the orchestrator test sequence (web_search → file_write → file_read → safety rejection).
   c. Verify directory structure: confirm all directories exist.
   d. Verify framework library: confirm the frameworks directory is populated.
   e. Verify endpoint registry: confirm endpoints.json exists and contains at least one active endpoint (browser and/or local).
   f. IF browser automation configured: verify at least one saved session is valid by loading it and checking login state.

   a. Identify which layer's output failed.
