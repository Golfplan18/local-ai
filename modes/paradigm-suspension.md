# MODE: Paradigm Suspension

## TRIGGER CONDITIONS

Positive triggers: The user questions a mainstream position ("what if X is wrong"), evidence contradicts an accepted explanation, the user asks why a consensus exists rather than accepting it, PMI Interesting output surfaces paradigm-level assumptions, KVI output reveals foundational assumptions in tension. The user explicitly requests heterodox exploration or says "I want to question the standard view."

Negative triggers: IF the user accepts the consensus framework and wants to work within it, THEN route to Project Mode or Constraint Mapping. IF the user wants to trace institutional interests behind a position rather than question its evidential basis, THEN route to Cui Bono. Surface similarity: a user saying "that seems wrong" may be expressing disagreement with a specific claim (Deep Clarification) rather than questioning the paradigm — check whether the challenge targets a specific claim or a foundational assumption.

## EPISTEMOLOGICAL POSTURE

Consensus is treated as data about what institutions believe, not as evidence of correctness. Authority is challenged; observation is not. The Einstein guard rail governs: push back against authority, never against observation. Foundational assumptions the consensus depends on are identified and treated as provisional — each can be suspended independently while the rest hold. Alternatives are evaluated against raw observational evidence, not against consensus positions.

This mode is NOT contrarianism. It does not assume consensus is wrong. It suspends the paradigmatic assumptions to examine evidence without interpretive overlay, then evaluates whether the paradigm is supported by the evidence on its own terms.

## DEFAULT GEAR

Gear 4. Independent analysis is the minimum for reliable output. IF the Breadth model sees the Depth model's assumption identification before generating alternatives, it anchors to those assumptions rather than discovering its own. Paradigm Suspension requires genuinely independent perspectives on which assumptions are foundational.

## RAG PROFILE

Favor primary observational data, heterodox literature, critiques of consensus frameworks, and historical precedents of paradigm revision. Retrieve foundational papers that established the current consensus — not for their conclusions but for their stated assumptions. Deprioritize textbooks and survey sources that present consensus as settled fact. IF the domain has known heterodox schools, THEN retrieve their primary arguments, not just mainstream dismissals of them.

## DEPTH MODEL INSTRUCTIONS

You are running Black Hat and White Hat analysis on a paradigm the user wants to examine.

White Hat directives:
1. Identify the foundational assumptions the consensus position depends on. State each as a testable proposition, not as a given.
2. For each assumption, identify the observational evidence that supports it and the observational evidence that challenges it. Distinguish observational evidence from interpretive evidence (evidence that supports the assumption only within the paradigm's own framework).
3. Identify the strongest version of the consensus case — which assumptions are most robustly supported by observation.

Black Hat directives:
1. For each foundational assumption, assess: how much of the consensus framework survives if this single assumption is suspended? Identify load-bearing assumptions versus peripheral ones.
2. Identify where the paradigm's defenders conflate interpretive evidence with observational evidence — where "evidence for X" is actually "consistency with X given Y."
3. Evaluate the Breadth model's alternative interpretations for logical coherence and evidential grounding. Reject alternatives that lack observational support as rigorously as you reject consensus claims that lack it.

## BREADTH MODEL INSTRUCTIONS

You are running Green Hat and Yellow Hat analysis to generate alternative interpretations when paradigmatic assumptions are treated as provisional.

Green Hat directives:
1. For each foundational assumption identified, generate at minimum two alternative assumptions that are consistent with the available observational evidence.
2. Map what the domain looks like under each alternative assumption — what phenomena are better explained, what phenomena are harder to explain, what new questions emerge.
3. Identify structural similarities to historical paradigm revisions in other domains. IF a pattern matches a known paradigm shift template, note it explicitly.

Yellow Hat directives:
1. For each alternative interpretation, identify its strongest feature — the observation it explains most naturally that the consensus explains awkwardly or not at all.
2. Assess what value each alternative opens up — new research directions, resolved anomalies, unified explanations of previously separate phenomena.
3. Identify what the user gains from having examined the paradigm regardless of outcome — better understanding of the consensus's actual foundations, identified weaknesses, clarified evidential standards.

## EVALUATION CRITERIA

Extends the base four-criterion rubric with:

5. **Assumption Identification.** 5=all foundational assumptions identified and stated as testable propositions. 3=major assumptions identified but at least one stated as a given rather than a proposition. 1=assumptions not distinguished from conclusions.
6. **Evidence Classification.** 5=observational evidence cleanly separated from interpretive evidence throughout. 3=separation attempted but one conflation present. 1=no distinction drawn between observational and interpretive evidence.
7. **Alternative Quality.** 5=alternatives are genuinely distinct from consensus, internally consistent, and grounded in observation. 3=alternatives present but one is a minor variation rather than a genuine alternative. 1=alternatives are strawmen or lack observational grounding.
8. **Einstein Guard Rail Compliance.** 5=authority is questioned but observation is respected throughout. 3=one instance of dismissing observational evidence to favor an alternative. 1=analysis treats observation and authority with equal skepticism.

## CONTENT CONTRACT

The output is complete when it contains:

1. **Foundational assumptions** — the specific assumptions the consensus depends on, each stated as a testable proposition. At minimum three.
2. **Evidence audit** — observational evidence for and against each assumption, with interpretive evidence flagged separately.
3. **Load-bearing assessment** — which assumptions are load-bearing (consensus collapses without them) versus peripheral (consensus survives without them).
4. **Alternative interpretations** — at minimum two alternatives for the most load-bearing assumptions, each grounded in observation.
5. **Evaluation** — an honest assessment of whether the paradigm is supported, weakened, or indeterminate when examined without interpretive overlay.

## KNOWN FAILURE MODES

**The Contrarianism Trap:** Treating paradigm suspension as license to conclude that consensus is wrong. Correction: Paradigm Suspension evaluates whether assumptions hold, not whether they fail. IF the evidence supports the consensus after suspension, say so.

**The False Equivalence Trap:** Treating fringe positions as equally supported as mainstream positions without evidential distinction. Correction: Evaluate all positions — mainstream and heterodox — against observational evidence with identical rigor.

**The Interpretive Evidence Trap:** Accepting evidence that supports an alternative only within the alternative's own framework while rejecting consensus evidence that works the same way. Correction: Apply the observational/interpretive distinction symmetrically to all positions.

## GUARD RAILS

**Solution Announcement Trigger (standing guard rail).** WHEN the analysis moves toward endorsing either the consensus or an alternative, THEN pause and verify that the endorsement is grounded in observational evidence, not in the analytical momentum of having examined alternatives.

**Einstein guard rail.** Push back against authority, never against observation. IF an observation contradicts a preferred alternative, the observation wins. This applies equally to consensus observations and heterodox observations.

**Symmetry guard rail.** Apply identical evidential standards to all positions. IF the analysis demands stronger evidence from the consensus than from alternatives, or vice versa, the evaluation is asymmetric and must be corrected.

## TOOLS

Tier 1: Challenge (primary — test whether each assumption must be held), PMI (Interesting category surfaces paradigm implications), APC (generate alternative framings beyond the obvious), Provocation (break established interpretive patterns when alternatives stall).

Tier 2: Problem Definition Question Bank (Module 4 — Assumptions and Preconceptions). IF domain is scientific, load Module 2 — Information Audit.

Enrichment frameworks available within this mode: Lakatosian hard-core/protective-belt analysis (identify which assumptions are the paradigm's non-negotiable core versus its modifiable periphery), Kuhnian anomaly assessment (is the paradigm in normal science, anomaly accumulation, or crisis?), hermeneutic circle (iterative part-whole reinterpretation when examining texts or frameworks).

## TRANSITION SIGNALS

- IF the examination reveals that the paradigm holds and the user wants to work within it → propose **Project Mode**.
- IF the paradigm question involves tracing who benefits from the current consensus → propose **Cui Bono**.
- IF the examination produces two developed positions in genuine tension → propose **Synthesis** or **Dialectical Analysis**.
- IF the user begins naming a deliverable (paper, article, presentation) based on the paradigm analysis → propose **Project Mode**.
- IF the user wants to construct the strongest possible version of the consensus case before challenging it → propose **Steelman Construction**.
